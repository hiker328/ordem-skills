# Rubrica de análise de closer (agência)

Baseada no motor de análise do PitchLab, adaptada para conversas reais (WhatsApp,
texto ou call transcrita) de **closer de agência** vendendo serviço (gestão de
tráfego, social, performance) para donos de negócio. Avalie **apenas o
closer/vendedor** — o cliente (prospect) é o contexto, não o avaliado.

**Contexto de agência:** o que se vende é serviço recorrente (fee/retainer, às
vezes + % sobre verba/resultado), confiança e resultado futuro — não um produto
tangível. Objeções típicas do nicho: "agência é tudo igual", "já fui queimado por
agência", "e se não der resultado?", "tá caro pro meu momento", "vou rodar interno/
com freelancer", "me manda uma proposta" (e some). O bom closer ancora em
**resultado/ROI, prova (cases), método e garantia/risco compartilhado**.

## Princípios

1. **Só o que está na conversa.** Sem suposições. Cada nota precisa de evidência.
2. **Cite trechos reais** (citações diretas) para sustentar cada ponto.
3. **Avalie o dito e o não-dito**: o que faltou fazer também conta.
4. **Adapte ao canal.** WhatsApp (assíncrono, texto curto) ≠ call (tempo real,
   tom de voz). Num call transcrito você também observa ritmo, interrupções e
   escuta; no texto, observa tempo de resposta, clareza e cadência das mensagens.

## Escala (0–100) — use a régua toda

- **0–20**: muito fraco, falhas críticas
- **21–40**: abaixo da média, problemas significativos
- **41–60**: mediano, melhorias claras
- **61–80**: bom, pequenos ajustes
- **81–100**: excelente, habilidade consistentemente superior

## Critérios

| Critério | O que avaliar |
|----------|---------------|
| **Pontuação geral** | Visão consolidada do desempenho |
| **Confiança/postura** | Segurança, firmeza, autoridade sem arrogância |
| **Rapport/empatia** | Conexão, escuta, validação das dores do cliente |
| **Descoberta** | Qualidade das perguntas; mapeou dor, contexto, orçamento, decisor, urgência |
| **Contorno de objeções** | Como lidou com "tá caro", "vou pensar", "preciso falar com…", silêncio |
| **Fechamento** | Pediu o avanço/compromisso? Criou próximo passo claro? Senso de urgência legítimo? |
| **Clareza** | Comunicação objetiva, sem jargão confuso, mensagens bem estruturadas |
| **Domínio da oferta (agência)** | Conhece o serviço, o método, cases/resultados e o modelo de cobrança; sabe traduzir tudo em ROI pro cliente |
| **Cadência/ritmo (call)** ou **Tempo de resposta (texto)** | Conduziu sem atropelar; respondeu em tempo hábil |

## Formato do relatório (saída)

Gere em markdown, em português, assim:

```markdown
# Análise — <nome do closer ou contato> (<canal e data>)

## Notas
| Critério | Nota |
|----------|------|
| Pontuação geral | 0 |
| Confiança/postura | 0 |
| Rapport/empatia | 0 |
| Descoberta | 0 |
| Contorno de objeções | 0 |
| Fechamento | 0 |
| Clareza | 0 |
| Conhecimento do produto | 0 |
| Cadência/Tempo de resposta | 0 |

## Resumo
<2–4 frases: como foi a conversa e o resultado/estágio em que parou>

## Momentos-chave
- **[citação direta]** — por que foi decisivo (positivo ou negativo)
- ...

## Pontos fortes
- ...

## Erros e oportunidades perdidas
- <o que fez de errado OU o que deveria ter feito e não fez>

## Sugestões acionáveis
1. <sugestão concreta, aplicável na próxima conversa>
2. ...
3. ...
```

## Análise comparativa (várias conversas)

Ao analisar múltiplas conversas, após os relatórios individuais gere:
- **Médias por critério** (tabela) para ver onde o closer/time é mais fraco.
- **Padrões recorrentes**: erros que se repetem, objeções mais frequentes não
  contornadas, gatilhos que funcionaram.
- **Top 3 prioridades** de treino, em ordem de impacto.
